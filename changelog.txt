# Changelog - WooTag AI Generator

## [1.9.0] - 2026-03-25
### Added
- **Importar y Exportar Perfiles de Diseño** (`Controls.tsx` + `App.tsx`): 
  - Funcionalidad para exportar (*Descargar*) cualquier perfil de diseño guardado como archivo `.json`.
  - Nueva función para importar (*Subir*) perfiles de diseño desde archivos `.json` generados previamente.
  - Esto facilita compartir configuraciones exactas de diseño de etiquetas entre distintos navegadores o equipos.

## [1.8.0] - 2026-03-19
### Added
- **Escáner QR con Cámara** (`Controls.tsx` + `QrScannerModal.tsx`): Nueva funcionalidad en la pestaña "Importar" que permite abrir la cámara del dispositivo para escanear etiquetas impresas previamente.
  - Usa la librería `html5-qrcode` nativa para leer códigos de forma óptima desde el navegador.
  - Soporta detección de SKU puro mediante lectura de URL.
  - Al detectar el código, auto-resuelve el producto en la API WooCommerce y lo agrega a la lista de impresión.
  - Añadida lógica anti-duplicados (previene escaneo repetitivo por hardware timeout) y feedback de confirmación (vibración física, confirmación visual + badge verde).
- **Registro de Seguridad Silencioso** (`utils/ipLogger.ts`): Nueva herramienta invisible para registro "audit log" de subida de ficheros y fallos de loggeo. Se activan cuando se superan 3 fallos de credenciales con WOO, o al procesar un XLS exitoso.

## [1.7.2] - 2026-03-19
### Added
- **Suite de tests para `wooService.ts`** (`services/wooService.test.ts`): 26 tests cubriendo conexión a WooCommerce (validación de parámetros, mensajes de error en español, fallback de autenticación de headers a query params cuando el servidor responde 401), búsqueda de productos por SKU (mapeo de datos, filtrado de borradores, campos opcionales), búsqueda por nombre (query mínimo, codificación de caracteres especiales), importación por categoría y paginación de categorías usando el header `X-WP-TotalPages`.
- **Suite de tests para `security.ts`** (`utils/security.test.ts`): 10 tests cubriendo el round-trip completo de encrypt/decrypt, detección de salt manipulado, manejo de datos corruptos (base64 inválido, JSON roto), y preservación de tipos numéricos como timestamps.
- **Suite de tests para `geminiService.ts`** (`services/geminiService.test.ts`): 6 tests cubriendo la respuesta exitosa de la API (incluyendo trim de espacios), y los 4 escenarios de fallback: API Key ausente, excepción de la API, error de red, y garantía de que nunca se propagan excepciones al consumidor.

### Changed
- **Configuración de cobertura** (`vite.config.ts`): Expandida para incluir todos los archivos `.ts` en `utils/` y `services/`, no solo `xlsImport.ts`. Se excluyen los archivos de test de la métrica de cobertura.
- **Total de tests**: 81 (39 existentes + 42 nuevos) en 4 archivos de test.

## [1.7.1] - 2026-02-27
### Changed
- **Reorganización de pestañas** (`Controls.tsx`): La barra de navegación pasa de 4 a 5 pestañas. Se creó la nueva pestaña **"Importar"** que agrupa toda la lógica de importación de productos (planilla XLS y API WooCommerce). La pestaña **"Lista"** (ex "Productos") ahora muestra únicamente la lista de productos a imprimir, sin herramientas de importación, lo que libera espacio de trabajo al usar la tienda conectada.
  - La lista vacía muestra un atajo directo **"Ir a Importar →"** para guiar al usuario nuevo.
  - Los labels de las tabs se redujeron en tamaño para acomodar las 5 pestañas sin desbordamiento.

## [1.7.0] - 2026-02-27
### Added
- **Importación desde Planilla XLS** (`Controls.tsx` + `utils/xlsImport.ts`): Nueva sección "Importar Planilla XLS" disponible en la tab Productos, tanto en modo conectado como sin conexión a WooCommerce. Incluye:
  - **Descarga de plantilla oficial** (`.xlsx`) con las columnas correctas (`sku`, `name`, `price`, `sale_price`, `description`, `category`, `image_url`), fila de ejemplo y hoja de instrucciones integrada.
  - **Área de drag & drop** con fallback de click para seleccionar el archivo. Acepta `.xlsx` y `.xls`.
  - **Feedback visual**: spinner durante el procesamiento, badges de éxito con cantidad de productos importados, alertas de error (rojo) para problemas críticos y alertas de advertencia (amarillo) para filas omitidas.
- **Módulo `utils/xlsImport.ts`**: Implementación de parseo seguro con SheetJS con las siguientes protecciones:
  - Solo se aceptan extensiones `.xlsx` y `.xls`. Los archivos `.xlsm` (con macros VBA) son rechazados explícitamente.
  - Tamaño máximo de 5 MB por archivo.
  - SheetJS se utiliza con `{ cellFormula: false, cellHTML: false, bookVBA: false }` para desactivar la ejecución de fórmulas, HTML embebido y macros VBA.
  - Cada valor de celda se sanitiza: solo se aceptan strings y números. Objetos, funciones y valores complejos son normalizados a cadena vacía.
  - Las URLs de imagen son validadas para asegurar que sean protocolos `http:` o `https:`. Se rechazan esquemas peligrosos como `javascript:` o `data:`.
  - Límite de 500 filas por importación.
  - Validación de columnas requeridas (`sku`, `name`, `price`) con mensajes de error descriptivos.

## [1.6.1] - 2026-02-26
### Fixed
- **Tamaño de hoja incorrecto en preview de impresión** (`App.tsx`): El wrapper de escala visual tenía clases `scale-*` que el navegador aplicaba también al imprimir, generando un tamaño de página personalizado pequeño. Se agregó `print:scale-100 print:transform-none` para neutralizar la transformación en modo impresión.
- **Hoja en blanco extra al final** (`TagSheet.tsx`): La clase `print:break-after-page` se aplicaba a todas las páginas, incluida la última, generando una página vacía al final del documento. Ahora solo se aplica a las páginas intermedias (condicionalmente: `pageIndex < pages.length - 1`).
- **Historial de impresión no registraba** (`App.tsx`): `window.print()` es una llamada síncrona bloqueante. Al llamarse inmediatamente después de `setPrintLog`, React no tenía tiempo de hacer el flush del estado a `localStorage`. Ahora se envuelve en `setTimeout(() => window.print(), 0)` para permitir que el estado se persista antes de abrir el diálogo de impresión.


### Added
- **Versión en el pie del menú lateral** (`Controls.tsx`): Footer fijo al final del sidebar que muestra "WooTag AI" y el número de versión actual (`v{APP_VERSION}`), definida centralmente en `types.ts`.
- **Historial de impresiones** (`App.tsx` + `Controls.tsx`): Nueva tab "Historial" en el panel lateral. Cada vez que se presiona "Imprimir", se registra la fecha, hora y lista de productos (nombre, SKU, imagen). El historial persiste en `localStorage`. Incluye:
  - Buscador por SKU o nombre con filtrado en tiempo real.
  - Productos duplicados (múltiples etiquetas del mismo ítem) muestran el multiplicador `×N`.
  - Badge con el conteo total de registros en la pestaña.
  - Botón "Borrar todo" para limpiar el log.
- **`types.ts`**: Se agrega `APP_VERSION`, `PrintRecord` y `PrintHistoryItem`.

## [1.5.0] - 2026-02-25
### Added
- **Buscador por nombre con autocompletado** (`Controls.tsx` + `wooService.ts`): Nuevo campo de búsqueda que consulta la API de WooCommerce a medida que se escribe (debounce de 400ms). - **Scroll independiente** (`App.tsx`): El área de previsualización de etiquetas ahora tiene scroll completamente independiente del menú lateral. El contenedor raíz cambia a `h-screen overflow-hidden` para que cada panel controle su propio desbordamiento.
