# Changelog - WooTag AI Generator

## [2.0.7] - 2026-04-01
### Fixed
- **Sincronización en la nube fallaba silenciosamente** (`services/cloudProfiles.ts`): Al actualizar los productos en tiempo real, ciertos campos que venían nulos o vacíos desde la API de WooCommerce (como `salePrice`, `image`, o `category`) se representaban como `undefined` en el objeto `Product`. El SDK de Firebase rechaza terminantemente cualquier escritura que contenga `undefined`, lo que provocaba que la aplicación móvil cancelara silenciosamente la sincronización hacia la nube. Se añadió una "limpieza segura" transparente antes de llamar a Firestore, eliminando los campos `undefined` para que los productos se inserten correctamente y aparezcan al instante en la PC.

## [2.0.6] - 2026-03-28
### Added
- **Sincronización de productos en tiempo real entre dispositivos** (`App.tsx`, `cloudProfiles.ts`): Los productos de la lista de impresión ahora se sincronizan automáticamente en tiempo real entre PC y móvil cuando ambos están logueados con la misma cuenta de Firebase. Ya no es necesario el modo Companion (QR) para ver en la PC los productos escaneados desde el móvil.
  - El sistema usa `onSnapshot` de Firestore sobre el documento del usuario, reaccionando a cambios en menos de 1 segundo.
  - Se implementó un mecanismo anti-loop con `lastProductsDevice` (ID único por dispositivo/navegador generado una vez y persistido en `localStorage`) y un ref `skipNextCloudWrite` para evitar que el dispositivo que recibe una actualización la vuelva a escribir.
  - Los productos se escriben a Firestore con un debounce de 1.5 segundos para no saturar la base de datos con cada escaneo.
  - El sync de productos vía sala (modo Companion/QR) sigue teniendo prioridad cuando `activeRoomId` está activo.

## [2.0.5] - 2026-03-28
### Improved
- **Mensajes de error en `CloudLoginModal.tsx`**: Se reemplazó el mensaje genérico "Ocurrió un error. Verificá tus datos." por mensajes específicos para cada código de error de Firebase: `auth/operation-not-allowed` (proveedor deshabilitado), `auth/admin-restricted-operation` (registro no habilitado), `auth/network-request-failed` (sin conexión), `auth/too-many-requests`, `auth/user-not-found`, `auth/wrong-password`, y un fallback que muestra el código de error técnico en lugar de ocultarlo. Adicionalmente se loguea el error completo en consola para facilitar el debug.

## [2.0.4] - 2026-03-28
### Fixed
- **Versión desincronizada en la UI** (`types.ts`, `vite.config.ts`): La constante `APP_VERSION` estaba hardcodeada a `'2.0.0'` en `types.ts` sin actualizar manualmente con cada release. Se refactorizó para que sea inyectada automáticamente por Vite desde el campo `version` de `package.json` en tiempo de build (via `__APP_VERSION__`). **A partir de ahora, para actualizar la versión basta con modificar `package.json`.**

## [2.0.3] - 2026-03-27

El móvil no puede ser "host" de una sala — ese rol corresponde a la PC. Se usa un hook reactivo que escucha `resize` para manejar también rotaciones de pantalla.
- **Formulario de registro mejorado en `CloudLoginModal.tsx`**:
  - Nuevo campo "Confirmar contraseña" visible solo al crear cuenta, con animación de entrada.
  - Feedback visual en tiempo real: el borde se vuelve verde si las contraseñas coinciden, rojo si no.
  - Texto indicador bajo el campo con mensaje explícito ("✓ Las contraseñas coinciden" / "✗ Las contraseñas no coinciden").
  - Validación previa a la llamada a Firebase: si las contraseñas no coinciden o tienen menos de 6 caracteres, se muestra el error sin consumir la API.
  - Botones para mostrar/ocultar contraseña (👁) en ambos campos.
  - Los campos de contraseña se limpian al cambiar entre modo Login y Registro.

## [2.0.2] - 2026-03-27
### Fixed

Se agrega `.htaccess` con `Cache-Control: no-store` para `index.html` (el punto de entrada siempre debe ser fresco) y `max-age=31536000, immutable` para los assets con hash de Vite (son inmutables por definición). También incluye `mod_rewrite` para soporte de rutas SPA a futuro.

## [1.7.0] - 2026-02-27
### Added
- **Importación desde Planilla XLS** (`Controls.tsx` + `utils/xlsImport.ts`): Nueva sección "Importar Planilla XLS" disponible en la tab Productos, tanto en modo conectado como sin conexión a WooCommerce. Incluye:
  - **Descarga de plantilla oficial** (`.xlsx`) con las columnas correctas (`sku`, `name`, `price`, `sale_price`, `description`, `category`, `image_url`), fila de ejemplo y hoja de instrucciones integrada.

  - **Feedback visual**: spinner durante el procesamiento, badges de éxito con cantidad de productos importados, alertas de error (rojo) para problemas críticos y alertas de advertencia (amarillo) para filas omitidas.
- **Módulo `utils/xlsImport.ts`**: Implementación de parseo seguro con SheetJS con las siguientes protecciones:
  - Solo se aceptan extensiones `.xlsx` y `.xls`. Los archivos `.xlsm` (con macros VBA) son rechazados explícitamente.
  - Tamaño máximo de 5 MB por archivo.
  - SheetJS se utiliza con `{ cellFormula: false, cellHTML: false, bookVBA: false }` para desactivar la ejecución de fórmulas, HTML embebido y macros VBA.
  - Cada valor de celda se sanitiza: solo se aceptan strings y números. Objetos, funciones y valores complejos son normalizados a cadena vacía.
  - Las URLs de imagen son validadas para asegurar que sean protocolos `http:` o `https:`. Se rechazan esquemas peligrosos como `javascript:` o `data:`.
  - Límite de 500 filas por importación.
  - Validación de columnas requeridas (`sku`, `name`, `price`) con mensajes de error descriptivos.

## [1.6.1] - 2026-02-26
### Fixed
- **Tamaño de hoja incorrecto en preview de impresión** (`App.tsx`): El wrapper de escala visual tenía clases `scale-*` que el navegador aplicaba también al imprimir, generando un tamaño de página personalizado pequeño. Se agregó `print:scale-100 print:transform-none` para neutralizar la transformación en modo impresión.
- **Hoja en blanco extra al final** (`TagSheet.tsx`): La clase `print:break-after-page` se aplicaba a todas las páginas, incluida la última, generando una página vacía al final del documento. Ahora solo se aplica a las páginas intermedias (condicionalmente: `pageIndex < pages.length - 1`).
- **Historial de impresión no registraba** (`App.tsx`): `window.print()` es una llamada síncrona bloqueante. Al llamarse inmediatamente después de `setPrintLog`, React no tenía tiempo de hacer el flush del estado a `localStorage`. Ahora se envuelve en `setTimeout(() => window.print(), 0)` para permitir que el estado se persista antes de abrir el diálogo de impresión.


### Added
- **Versión en el pie del menú lateral** (`Controls.tsx`): Footer fijo al final del sidebar que muestra "WooTag AI" y el número de versión actual (`v{APP_VERSION}`), definida centralmente en `types.ts`.
- **Historial de impresiones** (`App.tsx` + `Controls.tsx`): Nueva tab "Historial" en el panel lateral. Cada vez que se presiona "Imprimir", se registra la fecha, hora y lista de productos (nombre, SKU, imagen). El historial persiste en `localStorage`. Incluye:
  - Buscador por SKU o nombre con filtrado en tiempo real.
  - Productos duplicados (múltiples etiquetas del mismo ítem) muestran el multiplicador `×N`.
  - Badge con el conteo total de registros en la pestaña.
  - Botón "Borrar todo" para limpiar el log.
- **`types.ts`**: Se agrega `APP_VERSION`, `PrintRecord` y `PrintHistoryItem`.
